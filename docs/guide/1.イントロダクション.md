### 1.1 この講座で解説すること

この講座では、**2026年時点で、AIプロダクト開発の現場で実際に使われている React SPA のライブラリ構成**を紹介します。

本講座では、実際のAIプロダクト開発現場での経験をベースに、
- 今、現場で選ばれているライブラリは何か
- なぜそのライブラリが選ばれるのか
- 現場でどのように利用されているのか

を、現場のエンジニア視点で解説していきます。

なお適切なアーキテクチャやライブラリは、プロジェクトの規模やプロダクト特性によって異なります。今回紹介する内容は、**あくまで「AI プロダクトの現場では、こうした構成がある」という一例**です。  
内容を参考にしながら、ご自身のプロダクト開発や学習に役立ててもらえればと思います。


### 1.2 AI時代のReact開発

近年の開発現場では、フロントエンドに対して次のような要件が強く求められます。

- **開発スピード**
- **仕様変更への柔軟な対応力**

特にAIプロダクトでは、モデルの進化やプロンプトの調整によって、UIに求められる機能や見せ方が日々変化します。「作ってみないと正解がわからない」という不確実性が高いため、**変更に強い構成**が不可欠です。

#### 従来の React 開発の課題

こうした状況下で、従来よく見られた次のような構成は、開発のボトルネックになりがちです。

- `useEffect` にデータ取得や状態管理などの処理が集中し、コードが複雑である
- 状態管理・通信・画面遷移の役割が、1ファイル内に混ざり合っている
- ローディングやエラー処理が画面ごとにバラバラで、統一感がない

その結果、現状の仕様を理解することが難しい、機能追加のたびにバグが生まれるなど、変更の難易度が高くなることがありました。

本講座では、こうした課題を解決するための **「変更に強い、かつ、開発スピードを高められる」React 構成** を紹介します。

### 1.3 この講座で扱うライブラリと構成の考え方

本講座では、以下のライブラリなどを組み合わせた構成を解説します。

| ライブラリ | 役割 |
|---|---|
| **React Router (Framework Mode)** | ルーティング・データ取得の基盤 |
| **TanStack Query** | 非同期データの管理・キャッシュ |
| **Axios** | HTTPクライアント |
| **Zustand** | グローバルステート管理 |
| **React Hook Form & Zod** | フォーム管理・バリデーション |
| **MUI (Material UI)** | UIコンポーネント |

それぞれのライブラリについて、単なる使い方の解説ではなく、
- **なぜこのライブラリを採用するのか**
- **実際にどのように利用するのか**

を、サンプルアプリの実装を通して具体的に説明します。

### 1.4 対象者・前提知識

この講座は、次のような方を想定しています。

- **React での開発経験がある方**
- なんとなく SPA を作ることができるが、現場で採用されているプラクティスを知りたい方
- AIプロダクトで採用されている React 構成を知りたい方
- 変化の速い開発現場でのアーキテクチャに興味がある方

**前提知識:**
- React の基本（関数コンポーネント、useState / useEffect の理解）
- TypeScript の基本（型定義のメリットの理解）

文法や細かな API の解説は最小限にとどめ、「ライブラリの役割」と「構成の意図」に焦点を当てます。

### 1.5 講座の進め方

各章の解説で使用するソースコードは、章ごとに **Gitブランチ** として用意しています。
章ごとに該当ブランチのソースコードを見ながら解説を読むことで、より理解度が高まると思います。

**ブランチ名の命名規則:**
`ch{章番号}-{節番号}/{トピック名}` の形式になっています。

例:
- `ch4-2/routing-setup` （4.2章の説明用）
- `ch6-3-2/axios-implementation` （6.3.2項の説明用）

基本的には、各章の冒頭で「この章ではブランチ `xxxx` を使用します」と案内しますので、それに従ってブランチを切り替えてください。

#### アプリケーションテンプレートとしての利用

本講座で作成するアプリケーションは、GitHubリポジトリで公開しています。
このリポジトリには、以下のモダンな技術スタックが設定済みです。

*   React Router (Framework Mode)
*   Axios
*   TanStack Query
*   Zustand
*   React Hook Form & Zod
*   MUI

そのため、**初期セットアップの手間を省き、すぐに開発を始められる「アプリケーションベース」** としても利用可能です。
学習用としてだけでなく、実際のプロダクト開発のスタートアップテンプレートとしてもご活用ください。
